Web Development

I'm no designer, but I've been around the block

What if I told you that by flick of a button this website could be deployed in any of three ways? Thats the power of Infrastructure as Code (IaC). Checkout out my github repo for the code for the website you see here and the underlying AWS infrastructure. The current deployment is shown in the architecture diagram above! I don't get paid to make websites, but I used to do it all the time in highschool for hackathons and coding competitions. Building this website out is a sort of guilty pleasure, back to simpler times.

The other two deployments explore VMs and containers and each hold their own as a realistic solution for website hosting. They are dramatically more complex and fun to build, with end to end encryption capabilities and CI/CD for code deployment. But I can host an S3 static website with world-wide CDN delivery for free, can't beat that.

ETL Pipeline

Data is like oil, easier to manage when there's a pipeline

Above is one of my deployments. But the best solutions come in threes! Check out my code. I'm always looking to find and build multiple solutions for a single problem. The clouds flexibility and wide range of services can sometimes give us too much artistic liberty in our solution architecting. By working three solutions it helps me understand which solution is the best measured by the complexity for the client, architectural management considerations, and total operating cost.

In this case this solution came out of a business use-case from a buddy of mine, he wanted to move his architecture to the cloud to increase the resiliency and scalability of his offering. Since then he's adopted the architecture described in the image above. P.S Not my favorite solution but certainly the easiest to understand/manage

My Homelab

A ragtag team of old equipment I've gathered over the years

(Not pictured in case it gets sent to a shittyhomelabs subreddit) Before I took my homelab down for my move to Tampa, my homelab was a K8s cluster composed of two Raspberry Pis and one Win 10 box which ran 3 Ubuntu Virtual Machines. I quickly learned that Raspberry Pi 2 did not like modern security requirements for WIFI, you can hardcode some of the attributes required but if the NIC goes to sleep or shuts off, it will refuse to negotiate with the router.

Now I'm looking to shift gears with my homelab. When I find a more permanent place to settle I'm hoping to splurge on some upgrades and mess around with IOT in my home! Let's hope I don't turn my house into a giant CTF competition of exploits and vulnerabilities.

Website Visitor Tracker

Whose been stalking your website?

Nobody likes a guessing game, that's why logging is critical to any architecture. That's why I built out a lambda deployment that tracks all the users that visit my website. Spoiler alert! I'm super popular with users in China and Russia.

Don't worry I've turned this off so you're getting off scott free. There is only so much data you can get from network packets and most of the traffic came from web crawlers from AWS, Google, and potentially malicious 3rd parties. Doesn't make for the most interesting read... But this could pair well with a session recorder for analyzing user actions on a website.

Architecture Comparison

To determine the optimal solution, I designed and evaluated three distinct ETL architectures, each with different trade-offs in cost, scalability, and operational complexity:

Serverless (Lambda)

Fully serverless ETL using AWS Lambda, EventBridge, and S3

Pros:
  • Zero server management
  • Pay-per-execution pricing
  • Auto-scaling built-in
  • Lowest operational overhead
Cons:
  • 15-minute execution limit
  • Cold start latency
  • Complex for long-running jobs
  • Limited memory/CPU control

Est. Monthly Cost: $120-180

Containerized (ECS)

Container-based ETL using Amazon ECS with Fargate

Pros:
  • No execution time limits
  • Better resource control
  • Portable and scalable
  • Ideal for batch processing
Cons:
  • Higher cost than Lambda
  • Container image management
  • More complex networking
  • Requires container expertise

Est. Monthly Cost: $250-350

Virtual Machines (EC2)

Schedule-based EC2 instances with dedicated host licensing

Pros:
  • Leverage existing licenses
  • Full OS-level control
  • Cost-effective with scheduling
  • Familiar deployment model
Cons:
  • Manual scaling required
  • Higher operational overhead
  • Patch management needed
  • Less elastic than serverless

Actual Monthly Cost: $180-220

The Solution

After comprehensive analysis and cost modeling, the VM-based ETL architecture was selected as the optimal solution. This decision was driven by:

Key Implementation Details

  • Scheduled Execution: EC2 instances automatically start/stop based on EventBridge schedules
  • License Optimization: Utilized client's existing dedicated host licenses, saving $800/month
  • Infrastructure as Code: Fully automated deployment using Terraform
  • Monitoring: CloudWatch dashboards for pipeline health and data quality metrics

Results & Impact

  • 40% cost reduction compared to initial serverless estimate
  • 99.5% uptime for scheduled ETL jobs
  • 2-hour processing time for monthly data loads
  • Zero data loss with S3-based checkpoint system

Technical Highlights

Automation

EventBridge schedules trigger EC2 start/stop with Lambda orchestration

Security

CJIS-compliant VPC with encryption at rest and in transit

Scalability

Horizontal scaling through Auto Scaling Groups for peak loads

Lessons Learned

Architecture Selection

The "best" architecture isn't always the most modern. Existing licenses and operational constraints can make traditional VM-based solutions more cost-effective than serverless alternatives.

Scheduled Workloads

EventBridge-triggered start/stop automation for EC2 instances provides significant cost savings for predictable batch workloads without sacrificing reliability.
